# Content Calendar

Status: draft  
Mode: review-before-publish

This calendar is the **lagged development-sync calendar**. It keeps videos connected to current AI Maker Lab development while protecting recording time from being overwhelmed by fast AI work.

For the monetization-forward AI Playground / Minecraft / Racing starter season from `docs/concept.md`, see [season-plan-ai-builder-lab.md](./season-plan-ai-builder-lab.md).

## Calendar principle

> Record last week’s clearest artifact, not this week’s active work.

AI development can move faster than publishing. The calendar therefore uses a one-week content buffer:

- This week’s development creates raw proof and dev notes.
- This week’s recording uses last week’s proof.
- Deep dives are delayed until a topic proves it deserves more time.

## Recommended cadence

| Day | Content operation | Time box |
|---|---|---:|
| Monday | Pick one topic from last week’s notes. Choose the clearest artifact. | 30 min |
| Tuesday | Write a 7-line script and gather clips/screenshots. | 30–45 min |
| Wednesday | Record screen-share recap in one sitting. | 45–60 min |
| Thursday | Edit only for clarity: cut pauses, add labels, move demo first. | 60–90 min |
| Friday | Package draft: title, thumbnail text, description, 2–3 shorts. | 45–60 min |
| Weekend | Review metrics/comments if available; save lessons for next topic choice. | 30 min |

If the week is busy, collapse the process:

```text
Block 1: choose artifact + write 7-line script
Block 2: record + rough edit + save as draft
```

## Weekly topic selection ritual

Use this every Monday.

```text
1. Open last week’s dev notes.
2. Circle every visible artifact: demo, screenshot, diagram, QA evidence, failure, agent output.
3. Remove topics that require heavy context rebuilding.
4. Score the remaining topics.
5. Pick the lowest-effort topic with enough visual proof.
6. Save everything else as shorts, future segments, or private notes.
```

Selection rule:

```text
Best weekly topic = useful lesson + visible proof + low recording effort.
```

Do not choose a topic only because it is the newest development work.
