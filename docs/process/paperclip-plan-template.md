# Paperclip issue `plan` — template

Copy into the issue **`plan`** document (or link this file from the issue) when a change needs company-visible planning beyond OpenSpec deltas.

## Context

- Problem statement and user impact
- Links: OpenSpec change `openspec/changes/<id>/`, related `[PREFIX-123](/PREFIX/issues/PREFIX-123)` issues

## Goals

- Bulleted outcomes (user-visible or measurable)

## Non-goals

- What we are explicitly not doing in this slice

## User flows

- Primary flows (happy path + notable edge)

## Acceptance criteria

- [ ] 3–7 testable bullets (mirror or extend OpenSpec scenarios)

## Data model / persistence touchpoints

- Surreal tables, migrations, or “no DB change”

## API / adapter touchpoints

- Routes, adapters, ports — or “none”

## Risks

- Ranked risks + mitigations

## Rollout

- Feature flag, migration steps, telemetry, “immediate ship”
