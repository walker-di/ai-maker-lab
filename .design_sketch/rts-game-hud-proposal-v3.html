Design Proposal

RTS V3: make the map readable before making the HUD pretty

This proposal treats the game itself as the primary interface. Terrain, ownership, pathing, threat, selection, projectiles, and production timing all get distinct visual language, while the HUD stays compact enough to keep the battlefield playable.

Game-first readability: unit silhouettes, selection rings, threat zones, rally lines, and fog edges are visible without relying on decorative chrome.
RTS decision rhythm: economy, alerts, selection, command card, production queue, and minimap sit in predictable scan zones.
State coverage: combat alert, build placement, multi-select, empty selection, disabled commands, progress queues, and pause review use the same layout frame.
