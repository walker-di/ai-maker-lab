# UI Marketing Gap Plan

## 1. UI problem framing

The Marketing Manager UI is functionally present for products/personas but still has three user-visible gaps from `.pi/gaps/migrate-marketing-products-personas-implementation-gap.md`:

1. Product cards do not provide a semantic link to product detail. `ProductCard.svelte` renders the product name as plain `CardTitle`, and the products route passes `onViewPersonas={() => goto(...)}` only to the lower action button.
2. Product detail date rendering is not defensive. `ProductDetailView.svelte` calls `new Date(product.createdAt).toLocaleDateString()` directly; after `SurrealProductRepository.update()` uses Surreal `update()` semantics, returned records can lose `createdAt`, producing `Invalid Date`.
3. Marketing UI strings are hardcoded across app routes and shared marketing components. The OpenSpec slice requires visible marketing labels/headings/actions/errors/empty states to read from Paraglide messages in en/es/pt/ja.

Planning constraints:
- Use Svelte 5 direct `$props()`/`$derived()` style already present.
- Keep app routes as composition/routing boundary.
- Shared UI currently imports no app-local Paraglide messages; directly importing `$lib/paraglide/messages.js` from `packages/ui` is a boundary risk even though requested.

## 3. Proposed component changes with specific file paths

### Product card link

`packages/ui/src/lib/marketing/products/ProductCard.svelte`
- Add optional props such as:
  - `detailHref?: string`
  - `onViewDetail?: () => void`
- Preserve existing `onViewPersonas` to avoid breaking callers.
- Render the title/name as an anchor when `detailHref` or `onViewDetail` or `onViewPersonas` exists:
  - Preferred: app passes `detailHref` so the anchor has a real `href`.
  - If a handler is present, use `onclick` only to call the handler; do not make a button look like a link.
  - Keep edit/delete buttons separate and stop propagation only if needed.
- Add translated aria label/title text for the anchor if useful.

`apps/desktop-app/src/routes/marketing/products/+page.svelte`
- Pass `detailHref={`/marketing/products/${product.id}`}` and `onViewDetail={() => void goto(...)}` to `ProductCard`.
- Decide whether the existing lower “Personas” button should still navigate to detail or be renamed/kept as product-scoped personas.

### Product detail Invalid Date

`packages/ui/src/lib/marketing/products/ProductDetailView.svelte`
- Add a tiny presentational guard, e.g. `formatDate(value?: string)` or `$derived` that checks `Number.isNaN(date.getTime())`.
- Display a localized fallback such as `m.marketing_date_unknown()` or omit the “Created …”/“Last updated …” phrase when missing.
- Replace hardcoded `Created {date}` and `Last updated {date}` with message functions that accept `{date}`.

`packages/domain/src/infrastructure/database/marketing/SurrealProductRepository.ts`
- Backend fix should accompany UI guard: replace `db.update()` with Surreal merge semantics or fetch existing record and preserve `createdAt`.
- Add/update a repository test proving `createdAt` survives update and `updatedAt` changes.

### i18n imports and usage

For app route Svelte files:
- Add `import { m } from '$lib/paraglide/messages.js';`.
- Use `m.marketing_*()` for `<svelte:head><title>`, headings, descriptions, button labels, aria-labels, empty states, and loading text.

For `packages/ui/src/lib/marketing/**/*.svelte`:
- Requested approach: add `import { m } from '$lib/paraglide/messages.js';` and replace visible literals.
- Risk: `packages/ui` is intended reusable and should not depend on app-local `$lib`. Safer alternative is pass localized strings from app into shared UI, or create a ui-level i18n injection/label map. If implementing the request literally, run `bun run check:desktop-app` and package checks to verify `$lib` resolution from workspace package.

## 5. Risks and dependencies

- **Package boundary risk:** `packages/ui` importing `$lib/paraglide/messages.js` couples the shared UI package to `apps/desktop-app`. This conflicts with the workspace intent that UI is reusable. If direct imports fail checks or break package reuse, prefer passing translated strings/label maps from app routes.
- **Paraglide generation:** after editing all locale JSON files, generated Paraglide output may need regeneration depending on project scripts. Run the existing desktop check command.
- **Surreal update semantics:** UI guard prevents “Invalid Date”, but persistence should still preserve `createdAt`. Product repo update is a backend dependency.
- **Pluralization:** current simple singular/plural strings (`feature(s)`, `Persona(s)`, `scene(s)`) should become message functions; verify Paraglide message format supports the desired parameterization or use separate singular/plural keys.
- **Dynamic page title:** `model.product ? `${model.product.name} · Marketing` : 'Marketing Product'` needs localized suffix/fallback without breaking SSR/client behavior.
- **Locale route verification:** browser evidence showed `/es`, `/pt`, `/ja` still English; after messages are added, verify real locale routes render translated text.
